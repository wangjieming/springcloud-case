### [第2讲 让服务提供者的服务跑起来](https://github.com/wangjieming/springcloud-case/blob/main/doc/2.%E8%AE%A9%E6%9C%8D%E5%8A%A1%E6%8F%90%E4%BE%9B%E8%80%85%E7%9A%84%E6%9C%8D%E5%8A%A1%E8%B7%91%E8%B5%B7%E6%9D%A5.md)我们已经介绍了让服务提供者跑起来，但服务提供者跑起来之后，如果没有服务消费者，那么服务提供者也只能提供http url直接给客户端（比如浏览器或者app）调用，这样分布式的价值（有流量一起扛）就没有体现出来，这一篇，我们将会把服务提供者改造一下，然后供消费者调用, 这里只需要简单两步就可以完成

## 步骤：

###  a.服务提供者（项目：springcloud-demo）改造
#### 前面的文章[2. 让服务提供者跑起来](https://github.com/wangjieming/springcloud-case/blob/main/doc/2.%E8%AE%A9%E6%9C%8D%E5%8A%A1%E6%8F%90%E4%BE%9B%E8%80%85%E7%9A%84%E6%9C%8D%E5%8A%A1%E8%B7%91%E8%B5%B7%E6%9D%A5.md)已经写好了被消费者服务调用的controller,如下
```
@RestController
@RequestMapping("/")
public class HttpProviderController {

    @RequestMapping(value = "/httpServerProvider", method = RequestMethod.GET)
    public String sayHello() {
        return "Hello World";
    }  
}
```
#### 这里只需要在配置文件application.properties配置应用名就可以，加入如下配置：
spring.application.name=provider

##### 注：为什么需要应用名，这里简单说一下，因为服务提供者是一个服务集群，这一群服务共用一个应用名，然后把应用名注册到注册中心（zookeeper），服务消费者连到注册中心之后，就会知道某个应用名下有多少个服务，然后消费的时候，就会根据一定的负载均衡策略进行调用这群服务中的某一个，然后这一群服务提供者就可以共同承担消费者的流量了。

#### 6.因为我们都是在同一部机器调试，所以配置端口为8888，防止跟zookeeper端口（8080）或者服务（8088）冲突，在文件application.properties写入如下配置即可：
server.port=8088

#### 7.启动类需要加入注解@EnableFeignClients，从而让Feign这个模块可以启动，Feign有什么作用，前面一讲大概讲了一下，可以看作对http一个封装，转发。具体对原理，以后再说。
```
@EnableFeignClients
@SpringBootApplication
public class ConsumerApplication {

    public static void main(String[] args) {
        SpringApplication.run(ConsumerApplication.class, args);
    }

}
```
#### 8.增加一个接口，这个接口就是feign消费端，用来消费服务提供者的服务
```
@FeignClient(value = "provider")
public interface FeignConsumerService {

    @RequestMapping("/callServiceProvider")
    String index(@RequestParam("uid") String uid);
    
}
```
#### 9.增加一个controller，这个controller就是提供http访问路径给http客户端的用户（浏览器，Postman或者app）
```
@RestController
public class ConsumerController {

    @Autowired
    private FeignConsumerService feignConsumerService;

    @RequestMapping("/index")
    public String index(@RequestParam("uid") String uid) {
        return feignConsumerService.index(uid);
    }

}
```
